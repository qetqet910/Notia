# 2026-04-07 작업 리포트 (v1.2.1 릴리즈)

본 문서는 2026년 4월 7일 수행된 Notia 프로젝트의 주요 업데이트 및 개선 사항을 상세히 기록합니다.

## 1. 지식 그래프 고도화 (Knowledge Graph Enhancements)

지식 그래프의 활용성을 극대화하기 위해 데이터 관리 및 렌더링 로직을 근본적으로 개선했습니다.

- **자동 링크 추출 (Automatic WikiLink Extraction)**:
    - 노트를 저장(`addNote`, `updateNote`)할 때마다 본문에서 `[[위키링크]]` 패턴을 실시간으로 분석합니다.
    - 추출된 대상 노트들의 제목(target)을 중복 없이 추출하여 `notes` 테이블의 전용 컬럼인 `links`에 저장합니다.
- **초기 로딩 속도 최적화**:
    - 기존에는 그래프를 그리기 위해 모든 노트의 전체 본문을 페칭한 뒤 클라이언트에서 파싱했습니다.
    - 이제는 `select` 쿼리 시 `links` 컬럼만 가져오므로, 노트를 직접 열어보기 전에도 그래프 상에서 완벽한 연결망을 즉시 확인할 수 있습니다.
- **로컬 스토리지 연동 (SQLite/IndexedDB)**:
    - Tauri(SQLite) 환경에서도 `links` 데이터를 안전하게 보관할 수 있도록 테이블 스키마를 업데이트하고, JSON 직렬화/역직렬화 로직을 `localDB.ts`에 반영했습니다.

## 2. 알림 설정 UX 리디자인 (Notification UX Redesign)

사용자 편의성을 높이기 위해 사전 알림 설정 방식을 전면 개편했습니다.

- **배지 리스트 관리 (Badge-based Management)**:
    - 설정된 알림 시간들을 시각적인 배지 형태로 나열하여 한눈에 파악할 수 있게 했습니다.
    - 개별 배지의 `X` 버튼을 눌러 즉시 삭제할 수 있는 직관적인 인터페이스를 제공합니다.
- **커스텀 분 단위 입력 지원**:
    - 기존의 선택지 대신 사용자가 직접 원하는 분(minute) 단위를 입력하여 추가할 수 있습니다.
    - 최대 10개까지 알림 시간을 유연하게 관리할 수 있도록 로직을 확장했습니다.
